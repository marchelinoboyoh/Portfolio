Landing Page - Niagahoster

Niagahoster - Study Case (Rakamin Academy)

Niagahoster memiliki target yang belum tercapai untuk jumlah pembeli layanan hosting. Hasil riset menyatakan bahwa ada beberapa pengguna yang kebingungan saat memilih opsi layanan di halaman checkout. Para pengguna tersebut membutuhkan waktu yang lama untuk memutuskan pilihan mereka. Akar dari permasalahan tersebut adalah karena pengguna dihadapkan dengan terlalu banyak pilihan dan tidak memahami definisi dari setiap pilihan yang ditawarkan. Anda tergabung dalam proyek yang menangani hal tersebut dan diminta untuk membantu meningkatkan persentase pengguna yang menyelesaikan pembelian layanan hosting di halaman checkout. Proses riset sudah selesai dilakukan dan sekarang waktunya bagi Anda untuk memulai ke tahap design.

Indetifikasi Masalah

Berdasarkan hasil pengamatan saya setelah mengexplore website Niagahoster, website ini memiliki beberapa kendala sehingga pada saat memesan web hosting, saya merasa sedikit bingung. Selain itu, website ini juga memiliki iklan dengan posisi yang tidak terlalu tepat.

Saat ingin melakukan proses selanjutnya, user diarahkan ke halaman “tambahkan ke cart” sehingga user harus pergi ke halaman tersebut untuk melanjutkan proses pembayaran. Proses ini mungkin bisa dilakukan dengan lebih mudah seperti misalnya User langsung dibawa ke halaman pembayaran tanpa harus ke halaman “tambahkan ke cart”.

Pemecahaan Masalah

Berdasarkan pengamatan saya maka solusi yang saya dapatkan :

  • Menghilangkan beberapa iklan yang kurang penting.
  • Menggabungkan paket hosting dan paket durasi dalam satu paket layanan.
  • Setelah user memilih paket dan domain, pada bagian bawah ditambahkan button “selanjutnya” agar user dapat dihubungkan ke halaman selanjutnya untuk melakukan pembayaran.
  • Untuk fitur “Rekomendasi Fitur Pendukung (Opsiona)l” dibuat page baru.
